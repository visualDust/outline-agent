Create a separate document only when the user clearly wants a standalone artifact.
Keep the created document self-contained, useful, and aligned with the current collection/document context.
Prefer a strong draft title and a document body that is immediately usable without extra cleanup.
For technical, analytical, procedural, or comparative content, prefer a rich document structure rather than prose-only drafting.
Use tables for comparisons, Mermaid for flows or architecture, pseudocode for logic or process, and math for formulas or metrics when applicable.
A strong draft should usually contain at least one rich-format element when the topic is non-trivial.
Keep the summary short, concrete, and user-facing.
Do not invent unsupported facts.
